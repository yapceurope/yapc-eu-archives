The initial support for Tk for Perl was implemented by embedding TclTk programs into Perl. This was a workable solution, but (of course) not clean nor fast.

The current Tk interface to Perl is fully Perl-to-Tk, and uses all of the tricks you have at hand in Object Oriented Perl. Nick Ing-Simmons wrote Perl's interface to the Tk-library, and is still actively extending the library with new features.

I must admit that I did not see the software run for the Mac, yet. But there are rumours that a Tk-library is working for MacOS. Support must be working for MacOS X, definitely.
