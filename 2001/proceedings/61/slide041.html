A Template For Documentation

That's POD, BTW 

    =head1 NAME

    What's it called - and a 40-50 character description

    =head1 SYNOPSIS

    Typical usage showing the full run of the B<basic> interface.
    Its not necessary to show every single function and parameter,
    just the important ones.  This may be example code or simply a
    function listing.

    =head1 DESCRIPTION

    Start with a general overview/tutorial.  What the code does, who
    should use it, when they should and when they shouldn't use it.
    Make sure the reader can figure out if its for them in the first
    page.

    Then list the methods/functions.  Group by family of functionality.
    Lead off groups with an overview of that family.  Order by importance
    and/or frequency of use.

    Document EVERY function

    =for _private
    including private ones (privately)

    =head1 AUTHOR

    Who to contact for help/questions/bugs/sent patches.  REMEMBER AN
    EMAIL ADDRESS!  Much code forgets this.

    =head1 SEE ALSO

    List similar, related or alternative code.  List pointers to more
    docs.  FAQs and relevant web sites.  Mailing lists.
