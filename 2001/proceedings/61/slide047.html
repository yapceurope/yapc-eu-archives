Code Ownership Considered Harmful

We like to own things. This causes problem when it comes to code.

  • Causes a bottleneck

  • "No one else will see this" is very tempting

  • What if they author is "hit by a truck"?

  • Job security through obscurity can set in

  • Over familiarity breeds peculiarity in code.

  • Simple pig-headedness and pride must often be overcome
