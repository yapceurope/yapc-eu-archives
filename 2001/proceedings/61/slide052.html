Code Reviews

A code review, basically, is when another programmer looks at your code and comments on it.

Why?

  • Things get weird if you work in a vacuum.

  • Sanity check

  • Two branes >= one.

  • If you catch problems early in a review, its much cheaper than debugging later.

  • Spreads knowledge around. Almost free training.
